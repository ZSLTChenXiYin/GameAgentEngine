# 游戏状态权威边界

**中文** | [**English**](./GAME_STATE_AUTHORITY_EN.md)

本文件保留为当前主规范文档。

## 1. 总原则

- 游戏侧是高频真值的唯一权威来源。
- Engine 负责世界建模、NPC 认知、记忆、关系、叙事和交互推理。
- Engine 不直接拥有高频真值；需要时通过异步接口向游戏侧查询。
- Engine 中保存的玩家/NPC/场景信息允许是语义镜像，但不能替代游戏侧真值。

## 2. 游戏侧权威数据

以下数据只能以 `gameagentworker` 本地状态为准：

- 玩家 HP、体力、金币、饥饿、异常状态
- 玩家背包、装备、物品数量、耐久、可用性
- NPC/玩家当前精确位置与场景占用
- 场景即时状态，例如门是否打开、柜台是否有人、房间当前成员
- 任务真实阶段、完成条件、冷却、奖励发放状态
- 交易库存、价格、战斗判定、掉落、移动阻挡等高频规则状态

## 3. Engine 侧建模数据

以下数据适合长期驻留于 Engine：

- 世界节点、地点节点、NPC 节点、玩家节点
- 组织、阵营、归属、社交关系
- 长期记忆、共享记忆、世界记忆
- 场景语义描述、地点层级、世界时间语义状态
- 玩家/NPC 的身份印象、近期行为摘要、叙事上下文

## 4. 语义镜像数据

以下数据可以同步到 Engine，但仅作为推理辅助，不作为真值：

- `player_001` 看起来负伤、富有、可疑、受欢迎
- 玩家最近向酒馆老板送过礼
- 场景最近发生过争执、骚动、威胁、交易失败
- 当前房间主题、群聊气氛、房间成员的可见情绪摘要

## 5. Engine 可按需查询的数据

以下数据在 Engine 推理中允许请求，但默认不直接注入 prompt：

- 玩家/NPC 当前 HP 或 HP 区间
- 玩家当前金币
- 玩家是否持有某关键道具
- 某 NPC/玩家是否在当前场景
- 当前房间成员与场景占用
- 某任务当前真实阶段
- 某场景的即时状态

## 6. 玩家节点原则

- 玩家应作为一个正式节点存在于 Engine 中，建议类型为 `player` / `player_actor`。
- 玩家节点用于承载身份、关系、记忆和叙事位置。
- 玩家节点不是玩家真值状态的来源。
- 玩家真值状态查询应回到游戏侧。

## 7. 执行原则

- 对话类自然语言输入可以直接进入 Engine 推理。
- 高风险行为不得由自然语言直接修改状态，必须先经过游戏侧规则校验。
- 结构化动作先在游戏侧落地真值，再由 Engine 生成 NPC 反应与叙事反馈。
